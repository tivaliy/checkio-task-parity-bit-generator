During last I3E Industrial Conference in Robo-city a new version of communication protocol R2HI 2.0 (Robot-to-Human Interface) was adopted. A new version of R2HI interface extend existing one in adding error detecting code feature. This feature allows to send data from robots that are located on a long distance from human settlements with higher level of reliability. During the data transmission some bits can be substituted due to external events (radiation impact, different noises etc.). In R2HI 2.0 simple parity bit check technique is proposed.

R2HI 2.0 interface specification:
1. Data package (message) represented as a list of int numbers (0 ≤ number ≤ 255):
[d1, d2, d3, ..., dn], where d1 - the first letter in message
2. Each decimal number is an ASCII-code of respective character in binary + parity bit
See example:

    Chr | Dec  |   Bin   |P|  Bin + P | Dec
    ---------------------------------------
    'P' |  80  | 1010000 |0| 10100000 | 160
    'y' | 121  | 1111001 |1| 11110011 | 243
    't' | 116  | 1110100 |0| 11101000 | 232
    'h' | 104  | 1101000 |1| 11010001 | 209
    'o' | 111  | 1101111 |0| 11011110 | 222
    'n' | 110  | 1101110 |1| 11011101 | 221

        Message = [160, 243, 232, 209, 222, 221]

You have to implement "R2HI 2.0 translator/analyzer" that translates received data package (list of int) to string message. Before the translation erroneous numbers must be removed from package (list). Erroneous means decimal value contains wrong (odd) number of '1' in a binary form:

'P' → 80 → 1010000 + 0 → 10100000 → ...10110000... → 10110000 (Erroneous, 4-th bit was inverted)
Notice: The number of bits' substitutions can be any odd number (1, 3, 5, ...)

How it works?
Input message: [144, 16, 210, 214]
Remove erroneous characters (binary): [10010000, 00010000, 11010010, 11010110]
Binary result (remove parity bit): [10010000, 11010010]
Decimal: [72, 105]
Message string (ASCII): "Hi"

Input: A list of int numbers

Output: Message as a string
